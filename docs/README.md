# 文档目录

本文档目录包含保险智能体评测系统的所有产品文档。

---

## 📄 文档清单

| 文档 | 格式 | 说明 | 更新频率 |
|------|------|------|----------|
| PRD.md | Markdown | 产品需求文档（主文档） | 每次需求变更 |
| PRD.docx | Word | 产品需求文档（Word版） | 每次需求变更 |
| CHANGELOG.md | Markdown | 需求变更记录 | 每次需求变更 |
| generate_word.py | Python | Word文档生成脚本 | 按需 |

---

## 📝 如何使用

### 查看需求文档

**Markdown 版本**（推荐开发者使用）：
```bash
# 使用 VS Code 或 Typora 打开
open PRD.md
```

**Word 版本**（推荐产品/管理使用）：
```bash
# 生成 Word 文档
pip install python-docx
python generate_word.py

# 打开生成的文档
open PRD.docx
```

### 记录需求变更

每次需求变更时，按以下步骤操作：

1. **编辑 CHANGELOG.md**
   - 在"待处理变更"区域添加新的变更记录
   - 使用提供的模板格式
   - 分配变更编号（格式：REQ-YYYY-MM-DD-XXX）

2. **更新 PRD.md**
   - 根据变更内容修改对应章节
   - 更新"版本历史"表格
   - 更新"最后更新"日期

3. **重新生成 Word 版本**
   ```bash
   python generate_word.py
   ```

4. **提交变更**
   ```bash
   git add .
   git commit -m "docs: 更新需求文档 - [变更简述]"
   ```

```

   - 架构图

7. 数据模型

9. 部署需求
```

## ⚠️ 注意事项

1. **Markdown 是主文档**：PRD.md 是需求的主文档，Word 版本是从 Markdown 生成的
2. **保持同步**：每次更新 Markdown 后，必须重新生成 Word 版本
3. **记录变更**：任何需求变更都必须在 CHANGELOG.md 中记录
4. **版本号规则**：
   - 主版本号：重大架构变更
   - 次版本号：功能新增/删除
   - 修订号：bug修复/文档更新
