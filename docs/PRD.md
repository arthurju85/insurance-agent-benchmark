## 2. 产品概述

### 2.1 产品背景

随着大语言模型在保险行业的应用，需要一个专业的评测平台来评估不同保险Agent的能力表现，确保其具备足够的专业知识、推理能力和合规意识。

### 2.2 产品目标

1. **标准化评测**: 建立统一的保险Agent能力评测标准
2. **多维度评估**: 从知识、理解、推理、合规、工具五个维度全面评估
3. **数据驱动**: 通过排行榜和历史数据分析Agent能力演进
4. **防污染设计**: 题目变异引擎防止Agent过拟合

### 2.3 目标用户

| 用户角色 | 使用场景 | 核心需求 |
|----------|----------|----------|
| 保险公司 | 评估内部Agent产品 | 了解产品能力边界、发现改进点 |
| 技术厂商 | 展示产品能力 | 获得权威评测认证、生成排行榜 |
| 监管机构 | 监督AI应用合规性 | 确保Agent符合行业规范 |
| 研究人员 | 学术研究 | 获取评测数据集、对比分析 |

#### 3.1.4 题目变异引擎 [已完成]

**需求描述**: 自动生成题目变体，防止数据污染和Agent过拟合。

**功能点**:
- [x] 数值变异（日期/金额/年龄）
- [x] 实体替换（人名/公司名/地名）
- [x] 句式重组（同义词替换）
- [x] Ground Truth 同步更新
- [x] 验证规则自动调整

**变异策略**:
| 策略 | 示例 | 约束条件 |
|------|------|----------|
| 日期变异 | 2024-01-01 → 2023-12-17 | 保持时间间隔不变 |
| 金额变异 | 10,000元 → 9,710元 | 变异幅度 ±20% |
| 实体替换 | 张某 → 李某 | 同一次变异保持一致 |
| 句式重组 | "因为...所以..." → "由于...因此..." | 语义不变 |

**验收标准**:
1. 同一母题生成 ≥3 个不同变体
2. 变体验证规则自动更新
3. 支持批量生成题集变体

**相关代码**:
- `backend/src/db/variation_engine.py`

- `GET /api/v1/admin/submissions` - 管理员审核列表

## 7. 数据模型

### 7.1 核心实体

#### Question（题目）

```python
{
  "question_id": "string",      // 题目唯一ID
  "dimension": "string",        // 评测维度
  "question_type": "string",    // 题目类型
  "difficulty": "string",       // 难度等级
  "title": "string",            // 题目标题
  "content": "string",          // 题目内容
  "context": "string",          // 上下文信息
  "ground_truth": {},           // 标准答案
  "validation_rules": {},       // 验证规则
  "score": 100.0,               // 分值
  "is_variant": false,          // 是否为变体
  "parent_id": "string"         // 母题ID
}
```

#### Evaluation（评测）

```python
{
  "evaluation_id": "string",    // 评测ID
  "agent_id": "string",         // Agent ID
  "agent_name": "string",       // Agent名称
  "status": "string",           // 状态
  "total_score": 0.0,           // 总分
  "overall_percentage": 0.0,    // 百分比
  "total_questions": 0,         // 题目总数
  "completed_questions": 0,     // 完成数
  "question_results": []        // 详细结果
}
```
